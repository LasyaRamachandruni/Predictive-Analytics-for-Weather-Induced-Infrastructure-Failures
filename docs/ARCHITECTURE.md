## ğŸ¯ Key Design Decisions

### 1. **Hybrid Architecture**
- **Why**: Combines temporal patterns (LSTM) with feature interactions (trees)
- **Benefit**: Captures both sequential dependencies and complex feature relationships

### 2. **Dual Data Format**
- **Tabular**: For tree-based models that excel at feature interactions
- **Sequences**: For LSTM that captures temporal patterns
- **Benefit**: Each model type uses optimal data representation

### 3. **Chronological Splitting**
- **Why**: Time series data must respect temporal order
- **Benefit**: Prevents data leakage and realistic evaluation

### 4. **Feature Engineering Strategy**
- **Lags**: Capture delayed effects (e.g., rain today â†’ failure tomorrow)
- **Rolling stats**: Capture trends and patterns over windows
- **Benefit**: Rich feature set that captures temporal dynamics

### 5. **Ensemble Blending**
- **Tabular blend**: 50% RF + 50% XGB (combines strengths of both)
- **Hybrid blend**: 50% Tabular + 50% LSTM (combines temporal + feature-based)
- **Benefit**: Reduces overfitting and improves generalization
